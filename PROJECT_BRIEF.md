# Parallel — AI 世界引擎

> 构建一个你可以住进去的平行世界。

## 一句话定位

开源 AI 世界模拟引擎：自定义世界观 + 多角色自主行为 + 数字分身 + 持久记忆，让你活在另一个世界里。

## 项目目标

1. **简历作品**：展示 AI 工程 + 产品设计 + 全栈交付能力
2. **自用工具**：替代 Chai，解决卡顿/截断/无记忆/无世界观的痛点
3. **开源项目**：GitHub 开源，积累 star 和社区

## 核心功能（按优先级）

### P0 — MVP（1-2 周）
- [ ] **世界定义系统**：YAML/JSON 描述世界观（设定、规则、地理、历史）
- [ ] **多角色对话引擎**：场景中多个角色各自回应，不是 1v1
- [ ] **角色卡系统**：性格、目标、说话风格、关系网
- [ ] **持久记忆**：每个角色记住交互历史（向量数据库 + 摘要）
- [ ] **世界状态机**：时间流逝、关系变化、简单事件触发
- [ ] **Web UI**：对话 + 场景描述界面

### P1 — 第二阶段（+1 周）
- [ ] **关系系统**：好感度、亲密关系、敌对、师徒等关系类型及演进
- [ ] **世界事件引擎**：自动生成事件推动剧情（不只等玩家输入）
- [ ] **角色创建界面**：可视化创建/编辑角色

### P2 — 第三阶段（+1 周）
- [ ] **数字分身生成**：导入聊天记录 → 分析说话风格/性格 → 生成角色卡
  - 支持微信/WhatsApp/Telegram 导出格式
  - 需要对方授权（伦理考虑）
- [ ] **世界模板市场**：预制世界观（奇幻/科幻/现代/末日）

### P3 — 后续
- [ ] **语音**：TTS 输出（ElevenLabs / 本地 XTTS）
- [ ] **视觉**：角色头像生成、表情变化

## 起步策略：Fork vs 从零开始

不需要从零写引擎。三种可选路径：

| 方案 | 基础项目 | 优点 | 缺点 | 推荐度 |
|------|---------|------|------|--------|
| **A. Fork TaleWeave AI** | ssube/taleweave-ai (MIT) | 世界生成+NPC自主行为已实现，Python，最接近需求 | 偏 MUD 风格，需要加 UI 和关系层 | ⭐⭐⭐⭐ |
| **B. 用 NoizuRPG 框架** | noizurpg.com (Python) | 组件化设计，角色/世界/叙事/记忆解耦，API 式调用 | 早期项目，文档不全 | ⭐⭐⭐ |
| **C. 参考 AI Town 架构从零搭** | a16z-infra/ai-town (MIT) | JS/TS 全栈，Convex 实时引擎，可视化好 | 需要重写大量逻辑，偏"观察"不是"参与" | ⭐⭐⭐ |
| **D. 纯从零开始** | 无 | 完全自由，架构干净 | 最慢，1-2 周内很难出 MVP | ⭐⭐ |

**建议**：先花半天评估 TaleWeave AI 和 NoizuRPG 的代码质量，选一个作为基础 fork，在上面叠加你的差异化层（关系系统、数字分身、产品化 UI）。

## 技术选型建议

| 层 | 选择 | 理由 |
|---|------|------|
| 前端 | Next.js / React | 生态好，SSR 支持，vibe coding 友好 |
| 后端 | Python (FastAPI) | AI 生态最好，TaleWeave/NoizuRPG 都是 Python |
| LLM 后端 | OpenRouter / Ollama | 多模型切换，本地可跑 |
| 记忆 | SQLite + ChromaDB | 轻量持久化 + 向量搜索 |
| 世界状态 | SQLite | 事务支持，单文件部署 |
| 部署 | Docker | 一键启动 |

## 差异化

现有产品都缺少的组合：
1. **世界自己在动**（不只是等你说话）
2. **多角色同场景交互**（不是 1v1 聊天）
3. **数字分身**（从真人聊天记录克隆角色）
4. **关系系统**（亲密/友情/敌对，关系会演进）
5. **产品化体验**（现有开源方案 UI 都很糙）
