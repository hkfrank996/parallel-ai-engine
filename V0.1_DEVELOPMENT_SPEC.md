# Parallel v0.1 开发需求文档

> 版本目标：做出一个能玩的多角色 AI 场景对话 Demo。
> 核心体验：用户进入一个场景，说一句话，AI 导演决定哪些角色回应，角色基于各自身份、目标、关系和世界设定做出不同回应。
> 最后更新：2026-05-15

本文档记录了 v0.1 的**实际实现状态**，包含所有迭代修复。

## 技术栈（实际）

| 组件 | 选择 | 说明 |
|------|------|------|
| Framework | Next.js 15 + TypeScript | App Router |
| UI | React + Tailwind CSS v4 | `@tailwindcss/postcss` |
| 数据校验 | Zod | YAML 世界文件校验 |
| YAML 解析 | js-yaml | 世界定义 |
| 本地数据 | JSON 文件 | `data/store.json`，非 SQLite |
| LLM 调用 | 多 Provider 路由 + 三级 fallback | 用户配置 → 环境变量 → Mock |
| 包管理 | npm | |

### 为什么是 JSON 而不是 SQLite

`better-sqlite3` 需要 Visual Studio C++ build tools，用户环境不具备。JSON 文件读写简单，v0.1 的数据量（单会话、几十条消息）完全够用。v0.2 可迁移到 SQLite 或其他持久化方案。

### 多 Provider 路由 + 三级 Fallback

```
请求优先级：用户 UI 设置（localStorage）→ 环境变量（.env.local）→ Mock 模式
Fallback链：用户配置 API 失败 → 自动尝试环境变量 API → 最终 Mock
```

| Provider | 认证方式 | 适用场景 |
|----------|----------|---------|
| AnthropicProvider | `x-api-key` 或 `Authorization: Bearer`（自动检测） | Anthropic、Zhipu、LongCat 等兼容 API |
| OpenAIProvider | `Authorization: Bearer` | OpenAI 及兼容 API |
| MockProvider | 无 | 所有 API 失败时的兜底 |

**认证自动检测**：`AnthropicProvider` 根据 API URL 自动选择认证方式：
- `anthropic.com` / `bigmodel.cn` → `x-api-key` header
- 其他（LongCat 等）→ `Authorization: Bearer` header

**URL 路径自动补全**：所有 Anthropic 兼容 API 的 base URL 自动检测并补全 `/v1`，确保 `.../anthropic/v1/messages` 路径正确。

**API 参数**：`temperature: 0.8`（减少小模型重复），`max_tokens: 800`。

## v0.1 核心用户故事

### 用户故事 1：进入默认世界

作为用户，我打开应用后，看到 Neon Harbor 世界的场景、角色和世界规则。

### 用户故事 2：AI 导演驱动的互动

作为用户，我输入一句话后，**AI 导演**决定哪些角色回应（1-3 个，不是全部）。每个角色的回应体现：
- 自己的性格和目标
- 与其他角色的关系
- 当前场景和世界规则
- 情感真实性和角色一致性
- 对用户描述的叙事动作做出真实反应

### 用户故事 3：看到场景状态

作为用户，我能在侧边栏看到：当前世界、场景描述、角色卡片（含关系）、最近事件摘要、API 模式状态。

### 用户故事 4：配置 API

作为用户，我可以通过设置界面填入 API URL、API Key、模型名称，无需修改代码或环境变量。设置保存在浏览器 localStorage 中。

### 用户故事 5：中英双语切换

作为用户，我可以点击顶栏的"中文/EN"按钮切换角色回复语言。默认中文。

### 用户故事 6：保留对话记录

刷新页面后，对话记录不丢失（JSON 文件持久化）。

## v0.1 功能范围

### 已实现功能

1. **项目初始化**
   - Next.js + TypeScript + Tailwind CSS v4
   - `start.bat` 一键启动（安装依赖 → 启动服务 → 等待就绪 → 打开浏览器）

2. **世界定义系统**
   - `data/worlds/neon-harbor.yaml` — Demo 世界
   - Zod 校验 YAML 结构
   - `GET /api/world` 返回世界、会话、消息、事件

3. **角色卡系统**
   - 结构化字段：id, name, role, personality, goals, speaking_style, relationship_notes
   - 角色数据在 YAML 中定义

4. **AI 导演系统**（Director）
   - `lib/engine/director.ts`
   - AI 分析对话上下文，决定谁应该回应（1-3 人）
   - 考虑角色性格、目标、关系、当前场景
   - 理解叙事上下文：被移除/困住的角色不会被选中
   - 可选生成场景变化描述（sceneUpdate）
   - Mock 模式下跳过导演，所有角色都回应
   - 自带 fallback：Director API 失败 → 尝试环境变量 API → 最终所有角色

5. **多角色对话引擎**
   - 导演选择发言角色 → 为每个角色构建专属 prompt → LLM 生成回应
   - 角色回应基于：世界规则、场景、角色卡、关系网、最近对话
   - 叙事感知：角色接受用户描述的动作（攻击、忍术、空间术）为真实事件并做出反应
   - 三级 fallback：用户 API → 环境变量 API → Mock

6. **多 Provider LLM 路由**
   - `lib/llm/provider.ts` — 返回 primary provider + envFallback provider
   - `lib/llm/anthropicProvider.ts` — 通用 Anthropic 兼容（自动认证检测 + URL 补全）
   - `lib/llm/openaiProvider.ts` — OpenAI 兼容
   - `lib/llm/mockProvider.ts` — 每个角色 5 条预设回复（英文）

7. **API 设置 UI**
   - 设置弹窗：API URL、API Key（密码输入）、模型名称
   - 保存在 localStorage，每次请求传给后端
   - 顶栏显示当前 API 模式（Mock / 模型名称 + 绿色/琥珀色状态灯）

8. **中英双语切换**
   - 顶栏"中文/EN"按钮，默认中文
   - 语言偏好保存在 localStorage（`parallel-language`）
   - 角色 prompt 和 Director prompt 均根据语言设置生成对应语言指令
   - 中文模式下场景更新也使用中文

9. **Web UI**
   - 顶栏：应用名 + 世界名 + 类型标签 + 语言切换 + API 状态 + 设置按钮
   - 侧边栏：场景描述、角色卡片（含关系）、事件时间线、Mock 模式提示
   - 主区域：消息列表（角色名颜色编码）+ 输入框
   - Loading / Sending / Error 状态

10. **JSON 文件持久化**
    - `data/store.json` 保存 sessions、messages、events
    - 刷新后恢复对话

### 不做功能

v0.1 明确不做：

- 登录注册、多用户在线
- 角色创建器、世界编辑器
- 复杂长期记忆、向量数据库
- 数字分身、语音、头像生成
- 付费系统

## 数据模型

使用 JSON 文件 `data/store.json`，结构：

```typescript
interface StoreData {
  sessions: Array<{
    id: string;
    worldId: string;
    createdAt: string;
    updatedAt: string;
  }>;
  messages: Array<{
    id: string;
    sessionId: string;
    speakerType: "user" | "character" | "system";
    speakerId: string | null;
    speakerName: string;
    content: string;
    createdAt: string;
  }>;
  events: Array<{
    id: string;
    sessionId: string;
    summary: string;
    createdAt: string;
  }>;
}
```

## API 设计

### `GET /api/world`

返回世界配置、当前会话、所有消息和事件。

Response:
```json
{
  "world": { "id": "neon-harbor", "name": "Neon Harbor", ... },
  "session": { "id": "session-xxx" },
  "messages": [...],
  "events": [...]
}
```

### `POST /api/chat`

用户发言，AI 导演决定谁回应，返回角色回应。

Request:
```json
{
  "sessionId": "session-xxx",
  "message": "我发动时空忍术把 Mira 丢进异空间",
  "language": "zh",
  "llmConfig": {
    "apiUrl": "https://api.longcat.chat/anthropic",
    "apiKey": "ak-...",
    "model": "LongCat-Flash-Chat"
  }
}
```

`llmConfig` 为可选。不传则使用环境变量或 Mock 模式。`language` 为可选，默认 `"zh"`。

Response:
```json
{
  "userMessage": { "id": "...", "content": "..." },
  "characterMessages": [
    {
      "speakerId": "june",
      "speakerName": "June",
      "content": "..."
    }
  ],
  "event": {
    "summary": "雾气散去，Mira凭空消失..."
  },
  "sceneUpdate": "雾气突然散去，Mira凭空消失...",
  "degraded": false
}
```

`degraded: true` 表示使用了 fallback（不是用户配置的 API，而是环境变量 API 或 Mock）。

## 对话引擎逻辑

### AI 导演（Director）

新增的导演系统，在 API 模式下替代简单的"所有角色都回应"逻辑。

**工作流程**：
1. 构建 Director prompt（包含所有角色信息、对话历史、用户输入）
2. 调用 LLM，要求返回 JSON 格式：`{"speakers": ["char_id"], "sceneUpdate": "..."}`
3. 解析 JSON，验证角色 ID 存在性
4. 至少选择 1 个角色，最多 3 个
5. API 失败时自动尝试环境变量 fallback → 最终 fallback 到所有角色

**Director 决策依据**：
- 角色性格和目标 — 谁有动机回应？
- 角色关系 — 谁会针对谁说的话做出反应？
- 场景张力 — 有些情况下沉默比发言更有力
- **叙事感知** — 被用户动作移除/困住/失去能力的角色不会被选中
- 不总是让所有角色发言，营造自然的对话节奏

### `runTurn` 流程

```
用户输入
  → 保存用户消息
  → [API模式] 调用 Director（含 envFallback）决定谁发言 + 可选场景变化
  → [Mock模式] 所有角色发言
  → 为每个被选中的角色构建专属 prompt（含叙事规则+语言设置）
  → 调用 LLM 生成回应（失败时：尝试 envFallback → 最终 Mock）
  → 保存角色消息
  → 生成事件摘要（含场景变化）
  → 返回本轮结果（含 degraded 标记）
```

### Prompt 构建

每个角色的 System Prompt 包含：
- 世界名称、类型、规则
- 当前场景描述
- 角色卡：性格、目标、说话风格
- 角色关系网
- 其他在场角色

User Prompt 包含：
- 最近 8 条对话历史
- 用户最新输入
- 指令：以角色身份回应 + 对用户叙事动作做出真实反应

Prompt 要求：
- 第一人称，严格角色扮演
- 不替其他角色发言
- 2-4 句，简洁但有表现力
- 允许角色说谎、回避、保持沉默
- **叙事规则**：接受用户描述的动作为真实事件，做出对应反应
- **防重复**：每轮回复必须全新，不重复之前说过的话

## 迭代修复记录

### 2026-05-14/15 修复

| 问题 | 根因 | 修复 |
|------|------|------|
| 中文输入收到英文 Mock 回复 | Mock 模式无语言支持 | 添加语言参数贯穿整个链路（前端 → API → runTurn → prompt） |
| 3 个角色总是同时回复 | 无 Director 系统 | 创建 `director.ts`，AI 分析上下文选择 1-3 人 |
| 角色回复不自然/不连贯 | Prompt 缺少叙事规则 | 添加"叙事规则"：接受用户动作为真实事件并反应 |
| 角色重复自己说过的话 | 小模型 + 无 temperature + 无防重复指令 | 添加 `temperature: 0.8` + prompt 防重复指令 |
| LongCat API 404 | URL 缺少 `/v1` 路径 | 通用自动补全：任何 Anthropic 兼容 URL 自动加 `/v1` |
| LongCat 认证失败 | 代码硬编码 `x-api-key`，LongCat 需要 `Bearer` | 根据 URL 域名自动选择认证方式 |
| localStorage 有坏配置时直接跳 Mock | 无 fallback 链 | 实现 provider 三级 fallback：用户配置 → 环境变量 → Mock |
| degraded 标记永远 false | generateWithFallback 内部 catch 不传播 | 重构返回 `{ content, degraded }` 对象 |
| 默认语言英文 | 硬编码 `"en"` | 默认改为 `"zh"` |

## Demo 世界：Neon Harbor

```yaml
id: neon-harbor
name: Neon Harbor
tagline: A rainy cyberpunk port where everyone owes someone something.
genre: cyberpunk mystery

rules:
  - Technology is everywhere, but trust is rare.
  - Powerful factions control information.
  - Characters should speak with tension and hidden motives.

scene:
  id: rain-market
  name: Rain Market
  description: >
    A crowded night market under flickering neon signs...
```

3 个角色：Mira Voss（街头医生）、Ren Kaito（情报贩子）、June（失踪信使的学徒）。

核心冲突：一个 data courier 失踪了，三个人都知道更多却不肯说。

## 已知限制

1. **世界和角色是固定的** — 写在 YAML 中，运行时不能修改
2. **单会话** — 每个世界只有一个会话，不能新建/切换
3. **无记忆系统** — 只保留最近 12 条消息作为上下文
4. **小模型质量瓶颈** — LongCat-Flash-Chat 是小模型，长对话可能出现重复或上下文丢失。建议后续接入更大模型
5. **场景不真正变化** — Director 可以生成 sceneUpdate 但不会写回 YAML
6. **并发安全** — JSON 文件读写无锁，多个请求可能冲突（单用户场景影响不大）
7. **世界数据只有英文** — YAML 文件本身是英文，中文模式依赖 LLM 翻译
