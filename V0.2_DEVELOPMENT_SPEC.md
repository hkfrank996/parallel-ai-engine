# Parallel v0.2 开发需求文档

> 版本目标：角色真的记得发生过什么。
> 核心体验：第 10 轮对话能自然引用第 2 轮发生的事，关闭重启后记忆仍在。
> 最后更新：2026-05-15

## v0.2 核心用户故事

### 用户故事 1：角色会记住之前的对话

作为用户，第 10 轮对话时，角色自然引用第 2 轮发生的事，而不是"失忆"。

### 用户故事 2：看到记忆形成

作为用户，每轮对话后，我能看到"本轮记住了什么"——世界事实和角色记忆。

### 用户故事 3：侧边栏展示记忆

作为用户，我在侧边栏能看到：世界事实列表、每个角色卡片下方展示该角色的记忆。

### 用户故事 4：重启后记忆不丢失

关闭并重启应用后，世界事实和角色记忆仍然存在。

## v0.2 功能范围

### 已实现功能

1. **记忆数据模型**
   - `WorldFact`: 客观世界事实（"用户使用了忍术把 Mira 丢进异空间"）
   - `CharacterMemory`: 角色主观记忆，分类为 impression/conflict/affinity/secret/promise/event
   - 存储在 `data/store.json`，与 sessions/messages/events 并列

2. **记忆提取引擎** (`lib/engine/memoryExtractor.ts`)
   - 每轮对话后，用 LLM 分析本轮对话（用户输入 + 角色回复）
   - 自动提取：新世界事实（最多 3 条）、角色记忆（每角色最多 2 条）
   - 支持 fallback：用户 API → 环境变量 API → 跳过提取

3. **记忆检索** (`lib/engine/memoryRetriever.ts`)
   - 角色发言前，检索该角色相关的所有记忆
   - 同时获取最近的世界事实
   - 格式化为 prompt 友好的文本

4. **记忆注入 prompt**
   - `buildSystemPrompt` 新增 `memoryBlock` 参数
   - 角色的 system prompt 末尾注入 "MEMORY CONTEXT" 区块
   - 包含世界事实和角色个人记忆
   - 指令："Use these memories naturally — reference them when relevant"

5. **runTurn 集成**
   - 发言前：加载所有记忆 → 按角色过滤 → 注入 prompt
   - 发言后：提取新记忆 → 保存到 JSON
   - TurnResult 新增 `memoriesExtracted` 字段

6. **UI 记忆展示**
   - 侧边栏新增 "World Facts" 面板（蓝色标题，最多显示 5 条）
   - 角色卡片下方新增 "Memories" 区域（按类别标签，最多显示 4 条）
   - 消息区顶部新增蓝色提示条："本轮记忆" — 展示本轮新形成的记忆
   - `/api/world` 返回 worldFacts 和 characterMemories

### 不做功能

v0.2 明确不做：

- 向量数据库 / RAG 检索
- 复杂记忆压缩策略
- 多租户账号系统
- SQLite 迁移（v0.2 继续用 JSON）

## 数据模型

在 v0.1 基础上新增：

```typescript
interface WorldFact {
  id: string;
  sessionId: string;
  fact: string;            // 客观世界事实
  turnIndex: number;       // 第几轮产生
  createdAt: string;
}

interface CharacterMemory {
  id: string;
  sessionId: string;
  characterId: string;     // 哪个角色的记忆
  category: "impression" | "conflict" | "affinity" | "secret" | "promise" | "event";
  about: string;           // 关于谁/什么
  content: string;         // 记忆内容
  turnIndex: number;
  createdAt: string;
}
```

## 记忆提取 Prompt

```
Given a conversation turn, extract:
1. WORLD FACTS: objective facts about what happened (events, actions, changes)
2. CHARACTER MEMORIES: subjective observations from each character's perspective

Respond with JSON:
{
  "worldFacts": ["fact 1", "fact 2"],
  "characterMemories": [
    {"characterId": "id", "category": "...", "about": "...", "content": "..."}
  ]
}
```

限制：世界事实最多 3 条，角色记忆每角色最多 2 条。只提取有意义的、非显而易见的信息。

## 记忆在角色 Prompt 中的展示

```
MEMORY CONTEXT:
WORLD FACTS (things that happened):
  1. The user used space-time ninjutsu to trap Mira and Ren
  2. The market was destroyed by magic

YOUR MEMORIES:
  [affinity] about user: Mira wanted to protect the user from danger
  [conflict] about ren: Ren tried to sell information about Mira's location

Use these memories naturally — reference them when relevant.
```

```

→ 3 个角色回应（Mira, Ren, June），各有个性和立场
→ 提取 2 条世界事实 + 6 条角色记忆

```

## 已知限制

1. **记忆数量会增长** — 没有压缩或遗忘机制，长期对话后记忆条目可能过多
2. **提取增加延迟** — 每轮额外一次 LLM 调用（约 2-3 秒）
3. **记忆精度** — 小模型可能提取不够准确，大模型效果更好
4. **无记忆检索排序** — 目前只是取最新的 N 条，未来可加入相关性排序
5. **JSON 存储** — 数据量大时性能会下降，v0.3 可考虑 SQLite
