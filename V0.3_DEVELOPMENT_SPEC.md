# Parallel v0.3 开发需求文档

> 版本目标：这个世界不再只是"角色记住你说的话"——它会因为你而改变。
> 核心体验：角色翻脸、事件爆发、时间流逝、情绪演变——一个活的世界。
> 最后更新：2026-05-16

## v0.3 核心用户故事

### 用户故事 1：角色之间的关系会变化

作为用户，我威胁 Ren 后，他对我的信任下降、敌意上升。这种变化在 UI 上可见，且影响后续对话态度。

### 用户故事 2：世界会自己发生事件

作为用户，当我沉默或离开几轮回来，世界发生了新事件——通讯被干扰、监控被篡改、秘密被揭露。

### 用户故事 3：时间在流逝

作为用户，我看到从夜晚到黎明、从第 1 天到第 2 天。时间影响场景氛围和角色行为。

### 用户故事 4：角色有情绪

作为用户，角色不只是"有性格"——他们此刻的情绪会根据关系、记忆、时间动态变化，影响说话语气。

### 用户故事 5：角色会反思

作为用户，角色不只是记住原始事实，还会形成高层洞察——"Ren 似乎在把人命当价码"。

### 用户故事 6：时间线回放

作为用户，我在 Timeline 选项卡里能看到"这个世界怎么变成现在这样"——事件、事实、变化按时间排列。

## v0.3 功能范围

### 已实现功能

1. **世界时间引擎** (`lib/engine/worldTime.ts`)
   - day counter + 时间段（dawn / morning / afternoon / night）
   - 每 4 轮对话推进一个时段，4 个时段 = 1 天
   - `timeAtmosphere()` 为每个时段生成氛围文本，注入角色 prompt
   - `formatWorldTime()` 格式化显示

2. **关系系统** (`lib/engine/relationshipEngine.ts`)
   - 角色间关系维度：trust(0-100) / hostility(0-100) / dependency(0-100)
   - LLM 每轮分析对话，输出关系变化（delta -20 ~ +20）
   - 初始关系从 YAML `initial_relationships` 字段加载
   - 值域 clamp 到 0-100
   - 支持 language 参数，中文模式下 reason 用中文

3. **动态事件引擎** (`lib/engine/eventEngine.ts`)
   - 事件类型：plot_twist / character_action / environment / revelation / escalation
   - 触发条件：高敌意(60+)、信任崩塌、玩家接近秘密、叙事停滞
   - 每轮最多 1 个事件，事件必须有后果影响后续对话
   - 支持 language 参数，中文模式下 description/impact 用中文

4. **情感状态引擎** (`lib/engine/emotionalState.ts`)
   - **纯计算，无额外 LLM 调用**
   - 从关系数据 + 记忆模式 + 时间段 + 性格 → 推导角色当前 mood + arousal
   - 7 种 mood：烦躁 / 不安 / 温和 / 警惕 / 低落 / 平静 / 紧绷
   - arousal(0-100) 影响反应强度
   - 时间段修正：夜晚更警惕，上午更放松
   - 性格偏向：多疑角色倾向 suspicious，暴躁角色倾向 angry
   - 计算结果注入角色 system prompt 的"你现在的心理状态"区块

5. **角色反思**
   - `CharacterMemory` category 新增 `reflection` 类型
   - 记忆提取 prompt 增加 reflection 引导：角色从累积记忆中形成高层洞察
   - 反思类记忆在检索时优先展示（`memoryRetriever.ts` 先取 reflections 再取 others）

6. **总导演升级** (`lib/engine/director.ts`)
   - 接收关系数据 + 时间数据 + 事件历史
   - **叙事张力追踪**：calm → simmering → tense → breaking_point
   - **停滞度检测**：太久没事件时，提示 Director 主动制造冲突
   - 张力和停滞度作为上下文注入 Director prompt
   - 角色状态追踪（active / incapacitated），incapacitated 角色不能发言

7. **UI 增强**
   - 侧边栏改为 **World / Timeline 双选项卡**
   - **Timeline 选项卡**：时间线格式展示 session events + world events + world facts，按时间倒序排列，带类型图标和颜色
   - **Wait 按钮**：底部输入框右侧新增 Wait 按钮，点击发送"……"推进时间
   - **Turn Notice 面板**：每轮对话后顶部显示时间变化、关系变化、世界事件
   - **Provider 状态修正**：Header 从读 localStorage 改为读服务器实际状态，正确显示 OpenAI 或 Mock

8. **中文化**
   - 所有引擎（event / memory / relationship / director / prompt）支持 language 参数
   - 中文模式下输出全中文：事件描述、记忆内容、关系原因、场景更新、事件摘要
   - 记忆提取 prompt 特别强调：区分 user（玩家）和 character，正确归因动作发起者

### 不做功能

v0.3 明确不做：

- 复杂战斗系统
- 开放世界地图
- 多用户实时协作
- 角色自主规划（planning 层）—— v0.4 考虑
- 向量数据库 / RAG 记忆检索
- SQLite 迁移

## 数据模型

在 v0.2 基础上新增：

```typescript
// 世界时间
type TimeOfDay = "dawn" | "morning" | "afternoon" | "night";

interface WorldTime {
  sessionId: string;
  day: number;           // 第几天
  timeOfDay: TimeOfDay;  // 当前时段
  turnCount: number;     // 总轮数
}

// 角色关系
interface Relationship {
  id: string;
  sessionId: string;
  fromId: string;        // 谁对谁
  toId: string;
  trust: number;         // 0-100
  hostility: number;     // 0-100
  dependency: number;    // 0-100
  reason?: string;       // 最近一次变化的原因
  turnIndex: number;
  updatedAt: string;
}

// 世界事件
interface WorldEvent {
  id: string;
  sessionId: string;
  type: "plot_twist" | "character_action" | "environment" | "revelation" | "escalation";
  description: string;
  impact: string;
  turnIndex: number;
  createdAt: string;
}

// CharacterMemory.category 新增
category: "impression" | "conflict" | "affinity" | "secret" | "promise" | "event" | "reflection";
```

## 情感状态计算

```
输入：角色 + 关系列表 + 最近记忆 + 时间段 + 语言
输出：{ mood, arousal, description }

计算流程：
1. 基线：从平均 hostility / trust 计算 angry / warm / suspicious 分数
2. 记忆修正：最近冲突数 * 15 加分到 angry/tense，秘密数 * 20 加分到 afraid
3. 时间修正：night → suspicious+10, afraid+8 | dawn → afraid+10 | morning → warm+5
4. 性格修正：traits 含"暴躁" → angry+15 | traits 含"多疑" → suspicious+15
5. 取最高分 mood，计算 arousal = avgHostility*0.5 + conflictCount*15 + maxHostility>70?25:0
```

## 叙事张力计算

```
输入：关系列表 + 世界事件列表 + 当前轮数
输出：{ tension(0-100), stagnation(0-100), label }

tension = avgHostility*0.4 + (100-avgTrust)*0.2 + maxHostility>70?20:0
stagnation = min(100, turnsSinceLastEvent * 8)

label: <25 calm | <50 simmering | <75 tense | >=75 breaking_point
```

注入 Director prompt 的上下文：
- stagnation > 60 → "叙事停滞！需要制造冲突"
- tension > 70 → "张力极高！让事情发生"
- tension < 25 → "张力偏低，需要有人制造话题"

## runTurn 每轮流程

```
1. 推进世界时间 (advanceTime)
2. 初始化关系（首次）
3. 保存用户消息
4. Director 决策（接收 关系+时间+事件历史+张力+停滞度）
   → 输出：谁说话 + 场景变化 + 角色状态
5. 加载记忆（reflection 优先）
6. 计算情感状态（纯计算）
7. 逐角色顺序生成对话：
   - buildSystemPrompt（含 记忆+情感状态+氛围）
   - buildUserPrompt（含 其他人本轮已说的话）
   - generateWithFallback（provider → fallback → mock）
   - 后处理：去引号、压换行
8. 事件摘要（中文/英文）
9. 记忆提取（含 reflection 类别）
10. 关系分析（LLM 计算 delta）
11. 世界事件生成
```

每轮约 5-7 次 LLM 调用，耗时 40-55 秒。
